# M1.5 and M2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Execute one task at a time, run its focused tests, then run the repository quality gates before each commit.

**Goal:** Close the desktop runtime architecture, make token usage thread-aware, and add safe Codex user-input request/reply handling.

**Architecture:** `RuntimeController` owns the App Server, state aggregation, request registries, and snapshot publication without Electron window dependencies. `ServerRequestRegistry` connects protocol handlers to independent approval and input routers. The renderer receives only serialized snapshots and can invoke only explicit approval/input IPC methods.

**Tech Stack:** Electron 43, React 19, TypeScript 6, Vitest 4, Codex App Server 0.144.4.

## Global Constraints

- Keep `contextIsolation: true`, `nodeIntegration: false`, and `sandbox: true`.
- Do not persist user answers, tokens, command bodies, credentials, session files, or private paths.
- Do not add dependencies, copied third-party code, character assets, history rewrites, force pushes, or license changes.
- Preserve the distinction between mock/protocol verification and human end-to-end verification.
